Git & GitHub — Quick Starter

A tiny reference page with common commands and workflow for learning Git and GitHub.

Initialize a repo

Create a local repository and make your first commit:

git init
git add README.md
git commit -m "Initial commit"

Common commands

  • Check status: git status
  • See history: git log --oneline --graph --all
  • Create branch: git checkout -b feature
  • Switch branch: git checkout main
  • Merge: git merge feature
  • Stash changes: git stash

Connect to GitHub

Link local repo to a remote and push:

git remote add origin https://github.com/username/repo.git
git branch -M main
git push -u origin main

Typical GitHub flow

  1. Fork or clone the repo: git clone <url>
  2. Create a feature branch: git checkout -b my-feature
  3. Work, add, commit locally
  4. Push your branch: git push origin my-feature
  5. Open a Pull Request on GitHub and request review

Helpful files

  • README.md — project overview and instructions
  • .gitignore — files to exclude from commits
Tip: practice by creating a small repo, making commits, and pushing to GitHub.
